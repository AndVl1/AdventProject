# Сценарий 4: Просмотр статистики чтения

**Цель:** Проверить отображение статистики по книгам пользователя

**Предусловия:**
- Пользователь авторизован
- В базе данных есть книги с разными статусами
- Рекомендуется иметь минимум 3-5 книг с разными статусами

## Подготовка тестовых данных

Перед началом теста добавить книги:
1. 2 книги со статусом "Want to read"
2. 1 книга со статусом "Reading"
3. 2 книги со статусом "Read"

## Шаги

### 1. Открытие страницы статистики
1. Находясь на любой странице приложения, найти навигацию
2. Кликнуть на ссылку "Statistics" или перейти на `/stats`

### 2. Просмотр статистики
1. Дождаться загрузки данных
2. Проверить отображение всех метрик

## Ожидаемый результат

### Отображение метрик:
На странице должны отображаться 4 карточки со статистикой:

1. **Want to read** — количество книг со статусом "Want to read"
   - Ожидаемое значение: 2

2. **Reading** — количество книг со статусом "Reading"
   - Ожидаемое значение: 1

3. **Read** — количество книг со статусом "Read"
   - Ожидаемое значение: 2

4. **Total** — общее количество книг
   - Ожидаемое значение: 5

### Проверка через API:
```bash
# GET /api/books/stats
curl http://localhost:8080/api/books/stats \
  -H "Authorization: Bearer <JWT_TOKEN>"

# Ожидаемый ответ:
{
  "wantToRead": 2,
  "reading": 1,
  "read": 2,
  "total": 5
}
```

### Проверка расчётов:
- `total` = `wantToRead` + `reading` + `read`
- Все значения неотрицательные
- Статистика соответствует фактическому количеству книг пользователя

## Динамическое обновление

### После добавления новой книги:
1. Добавить книгу со статусом "Reading"
2. Перейти на страницу статистики
3. Значение "Reading" должно увеличиться на 1
4. Значение "Total" должно увеличиться на 1

### После изменения статуса:
1. Изменить статус книги с "Reading" на "Read"
2. Обновить страницу статистики
3. Значение "Reading" должно уменьшиться на 1
4. Значение "Read" должно увеличиться на 1

### После удаления книги:
1. Удалить книгу со статусом "Want to read"
2. Обновить страницу статистики
3. Значение "Want to read" должно уменьшиться на 1
4. Значение "Total" должно уменьшиться на 1

## Возможные ошибки

### Неавторизованный доступ:
- Если JWT токен отсутствует — перенаправление на `/login`

### Пустая статистика:
- Если у пользователя нет книг — все значения равны 0
- Страница должна корректно отображать нулевые значения

### Проблемы с сервером:
- Если backend недоступен — отображается сообщение об ошибке
- "Failed to load stats"

## Критерии успеха
- [x] Статистика корректно рассчитывается
- [x] Все метрики отображаются в UI
- [x] Значения соответствуют фактическим данным
- [x] Статистика обновляется при изменении данных
- [x] API возвращает корректные данные
- [x] Общая сумма равна сумме всех категорий
