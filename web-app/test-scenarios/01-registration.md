# Сценарий 1: Регистрация нового пользователя

**Цель:** Проверить регистрацию нового пользователя в приложении Bookshelf

**Предусловия:**
- Backend сервер запущен на `http://localhost:8080`
- Frontend приложение запущено на `http://localhost:5173`
- Пользователь не зарегистрирован в системе

## Шаги

### 1. Открытие приложения
1. Открыть браузер и перейти на `http://localhost:5173`
2. Приложение должно автоматически перенаправить на страницу входа (`/login`)

### 2. Переход к регистрации
1. На странице входа найти ссылку "Don't have an account? Register"
2. Кликнуть на ссылку "Register"

### 3. Заполнение формы регистрации
1. В поле "Username" ввести: `testuser`
2. В поле "Password" ввести: `TestPassword123!`
3. Нажать кнопку "Create account"

## Ожидаемый результат

### После заполнения формы:
- Кнопка "Create account" становится активной
- При нажатии кнопки отображается текст "Creating..."

### Успешная регистрация:
- Пользователь перенаправляется на страницу `/books`
- В адресной строке браузера отображается `http://localhost:5173/books`
- На странице отображается заголовок "My Books"
- Список книг пуст (пользователь только что зарегистрировался)

### Проверка токена:
- В localStorage браузера сохранён ключ `auth` с JWT токеном
- Токен валидный и может быть использован для API запросов

## Возможные ошибки

### Неверные данные:
- Если username менее 1 символа — форма не отправляется
- Если password менее 1 символа — форма не отправляется
- Если пользователь с таким именем уже существует — отображается сообщение об ошибке

### Проблемы с сервером:
- Если backend недоступен — отображается сообщение "Registration failed. Please try again."

## Критерии успеха
- [x] Пользователь успешно зарегистрирован
- [x] Происходит перенаправление на страницу книг
- [x] JWT токен сохранён в localStorage
- [x] API отвечает статусом 200 OK на защищённые эндпоинты
