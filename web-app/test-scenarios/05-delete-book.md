# Сценарий 5: Удаление книги из библиотеки

**Цель:** Проверить удаление книги из личной библиотеки

**Предусловия:**
- Пользователь авторизован
- В списке есть минимум одна книга
- Пользователь владеет книгой (является владельцем)

## Шаги

### 1. Подготовка к удалению
1. Открыть страницу `/books`
2. Найти книгу, которую нужно удалить
3. Запомнить id книги (для проверки через API)

### 2. Удаление книги
1. Найти кнопку "Delete" рядом с книгой
2. Кликнуть на кнопку "Delete"
3. Подтвердить удаление (если есть подтверждение)

## Ожидаемый результат

### Успешное удаление:
- Книга исчезает из списка на UI
- Список обновляется без перезагрузки страницы
- Количество книг в списке уменьшается на 1

### Проверка через API:
```bash
# GET /api/books не должен содержать удалённую книгу
curl http://localhost:8080/api/books \
  -H "Authorization: Bearer <JWT_TOKEN>"

# Удалённая книга отсутствует в ответе

# DELETE /api/books/{id}
curl -X DELETE http://localhost:8080/api/books/1 \
  -H "Authorization: Bearer <JWT_TOKEN>"

# Ожидаемый ответ: 204 No Content

# Повторный запрос GET /api/books/1
curl http://localhost:8080/api/books/1 \
  -H "Authorization: Bearer <JWT_TOKEN>"

# Ожидаемый ответ: 404 Not Found
```

### Проверка статистики:
1. Перейти на страницу `/stats`
2. Общее количество книг уменьшилось на 1
3. Статистика по статусам обновилась

## Возможные ошибки

### Неавторизованный доступ:
- Если JWT токен истёк — перенаправление на `/login`
- API возвращает 401 Unauthorized

### Книга не найдена:
- Если книга с указанным id не существует — API возвращает 404 Not Found
- Если книга уже удалена — повторное удаление возвращает 404

### Книга другого пользователя:
- Пользователь не может удалить чужую книгу
- API возвращает 404 Not Found (книга не найдена для текущего пользователя)

### Проблемы с сервером:
- Если backend недоступен — отображается сообщение об ошибке

## Тестовые сценарии

### Сценарий A: Удаление единственной книги
1. Пользователь имеет 1 книгу
2. Удалить книгу
3. Список становится пустым
4. Статистика показывает 0 книг

### Сценарий B: Удаление книги из середины списка
1. Пользователь имеет 5 книг
2. Удалить книгу #3
3. Книги #4 и #5 смещаются вверх
4. Идентификаторы остальных книг не меняются

### Сценарий C: Удаление книги с определённым статусом
1. Удалить книгу со статусом "Reading"
2. Статистика "Reading" уменьшается на 1
3. Общая статистика уменьшается на 1

- [x] API возвращает статус 204 No Content
- [x] Повторный запрос возвращает 404 Not Found
