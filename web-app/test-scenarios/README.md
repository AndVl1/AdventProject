# Пользовательские сценарии тестирования Bookshelf

Каталог содержит пользовательские сценарии для ручного и автоматизированного тестирования full-stack приложения Bookshelf.

## Структура сценариев

### Сценарий 1: Регистрация нового пользователя
**Файл:** `01-registration.md`

Проверяет процесс регистрации нового пользователя в приложении, включая:
- Переход со страницы входа на регистрацию
- Заполнение формы регистрации
- Сохранение JWT токена
- Перенаправление на страницу книг

### Сценарий 2: Добавление новой книги
**Файл:** `02-add-book.md`

Проверяет добавление книги в личную библиотеку:
- Заполнение формы добавления книги
- Валидация полей (title, author, status)
- Отображение добавленной книги в списке
- Проверка через API

### Сценарий 3: Изменение статуса книги
**Файл:** `03-change-status.md`

Проверяет изменение статуса книги при чтении:
- Изменение статуса с "Want to read" на "Reading"
- Изменение статуса с "Reading" на "Read"
- Обновление UI без перезагрузки страницы
- Проверка через API

### Сценарий 4: Просмотр статистики чтения
**Файл:** `04-statistics.md`

Проверяет отображение статистики по книгам пользователя:
- Отображение всех 4 метрик (Want to read, Reading, Read, Total)
- Корректность расчётов
- Динамическое обновление статистики
- Проверка через API

### Сценарий 5: Удаление книги из библиотеки
**Файл:** `05-delete-book.md`

Проверяет удаление книги из личной библиотеки:
- Удаление через UI
- Исчезновение книги из списка
- Обновление статистики
- Проверка через API

## Подготовка к тестированию

### Требования
- Java 21+
- Node.js 18+
- Maven 3.9+
- npm

### Запуск backend
```bash
cd web-app/backend
./mvnw spring-boot:run
```
Backend запускается на `http://localhost:8080`

### Запуск frontend
```bash
cd web-app/frontend
npm install
npm run dev
```
Frontend запускается на `http://localhost:5173`

## Проведение тестирования

### Ручное тестирование
1. Запустить backend и frontend
2. Открыть браузер на `http://localhost:5173`
3. Следовать шагам в каждом сценарии
4. Отмечать выполненные критерии успеха

### API тестирование
Для тестирования API можно использовать `curl`:
```bash
# Регистрация
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"TestPassword123!"}'

# Вход (получение токена)
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"TestPassword123!"}'

# Получение списка книг
curl http://localhost:8080/api/books \
  -H "Authorization: Bearer <JWT_TOKEN>"
```
