# Сценарий 2: Добавление новой книги

**Цель:** Проверить добавление новой книги в личную библиотеку

**Предусловия:**
- Пользователь зарегистрирован и авторизован
- Открыта страница `/books`
- Список книг пуст или содержит несколько книг

## Шаги

### 1. Открытие формы добавления
1. На странице `/books` найти форму добавления книги
2. Форма должна содержать поля:
   - Title (название книги)
   - Author (автор)
   - Status (выпадающий список)

### 2. Заполнение формы
1. В поле "Title" ввести: `The Great Gatsby`
2. В поле "Author" ввести: `F. Scott Fitzgerald`
3. В выпадающем списке "Status" выбрать: `Want to read`

### 3. Отправка формы
1. Нажать кнопку "Add Book"
2. Дождаться ответа от сервера

## Ожидаемый результат

### Валидация формы:
- Все поля обязательны для заполнения
- Если поле не заполнено — форма не отправляется
- Максимальная длина полей: Title — 255 символов, Author — 255 символов

### Успешное добавление:
- Книга появляется в списке ниже формы
- Книга отображается с введёнными данными:
  - Title: "The Great Gatsby"
  - Author: "F. Scott Fitzgerald"
  - Status: "Want to read"
- Книга получает уникальный id
- Поля формы очищаются после добавления

### Проверка через API:
```bash
# GET /api/books должен возвращать добавленную книгу
curl http://localhost:8080/api/books \
  -H "Authorization: Bearer <JWT_TOKEN>"

# Ожидаемый ответ:
[
  {
    "id": 1,
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "status": "WANT_TO_READ",
    "createdAt": "2026-04-16T10:30:00Z"
  }
]
```

## Возможные ошибки

### Неавторизованный доступ:
- Если JWT токен истёк или отсутствует — перенаправление на `/login`

### Неверные данные:
- Если title пустой — API возвращает 400 Bad Request
- Если author пустой — API возвращает 400 Bad Request
- Если status не выбран — API возвращает 400 Bad Request

### Проблемы с сервером:
- Если backend недоступен — отображается сообщение об ошибке

## Критерии успеха
- [x] Книга успешно добавлена в базу данных
- [x] Книга отображается в списке на UI
- [x] Данные книги корректны (title, author, status)
- [x] API возвращает статус 201 Created
- [x] Форма очищается после добавления
