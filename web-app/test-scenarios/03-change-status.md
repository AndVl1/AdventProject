# Сценарий 3: Изменение статуса книги

**Цель:** Проверить изменение статуса книги при чтении

**Предусловия:**
- Пользователь авторизован
- В списке есть минимум одна книга
- Книга имеет статус "Want to read" или "Reading"

## Шаги

### 1. Просмотр списка книг
1. Открыть страницу `/books`
2. Найти книгу со статусом "Want to read"
3. Запомнить id книги (обычно отображается в UI или можно посмотреть через DevTools)

### 2. Изменение статуса на "Reading"
1. Найти выпадающий список "Status" рядом с книгой
2. Выбрать статус "Reading"
3. Дождаться обновления данных

### 3. Изменение статуса на "Read"
1. Найти ту же книгу в списке
2. В выпадающем списке "Status" выбрать "Read"
3. Дождаться обновления данных

## Ожидаемый результат

### Изменение статуса на "Reading":
- Статус книги обновляется на "Reading"
- Изменение отображается немедленно (без перезагрузки страницы)
- В UI книга помечается как "Reading"

### Изменение статуса на "Read":
- Статус книги обновляется на "Read"
- Книга помечается как прочитанная
- В UI книга отображается в группе "Read"

### Проверка через API:
```bash
# GET /api/books/{id} должен возвращать обновлённый статус
curl http://localhost:8080/api/books/1 \
  -H "Authorization: Bearer <JWT_TOKEN>"

# Ожидаемый ответ после изменения на "Read":
{
  "id": 1,
  "title": "The Great Gatsby",
  "author": "F. Scott Fitzgerald",
  "status": "READ",
  "createdAt": "2026-04-16T10:30:00Z"
}

# PATCH /api/books/{id}/status
curl -X PATCH http://localhost:8080/api/books/1/status \
  -H "Authorization: Bearer <JWT_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"status": "READ"}'

# Ожидаемый ответ: 200 OK
```

## Возможные статусы
- `Want to read` — хочу прочитать
- `Reading` — читаю сейчас
- `Read` — прочитано

## Возможные ошибки

### Неавторизованный доступ:
- Если JWT токен истёк — перенаправление на `/login`

### Неверный статус:
- Если передан невалидный статус — API возвращает 400 Bad Request

### Книга не найдена:
- Если книга с указанным id не существует — API возвращает 404 Not Found
- Если книга принадлежит другому пользователю — API возвращает 404 Not Found

### Книга другого пользователя:
- Пользователь видит только свои книги
- Попытка изменить статус чужой книги возвращает 404

## Критерии успеха
- [x] Статус книги успешно изменён
- [x] Изменение отображается в UI без перезагрузки
- [x] API возвращает статус 200 OK
- [x] Данные в базе данных обновлены
- [x] Пользователь может изменять статус только своих книг
